# Quaeris Brand Fork Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split the single-brand white-label site into build-time brand packages — Veris stays the template (visually unchanged), Quaeris ships as the first real skin per `docs/superpowers/specs/2026-06-12-quaeris-brand-fork-design.md`.

**Architecture:** A tiny generator script (`scripts/select-brand.mjs`) writes `src/brand-active/{config.ts,fonts.ts,tokens.css}` as re-exports of the brand chosen by the `BRAND` env var (default `veris`). The app imports only from `@/brand-active/*` and the existing `@/config/brand.config` shim, so each build statically contains exactly one brand. *(This replaces the spec's bundler-alias idea with a generated module — same build-time outcome, zero dependence on Turbopack alias behavior, works identically for tsc, dev, and build.)*

**Tech Stack:** Next.js 16 (App Router, Turbopack), Tailwind v4 + CSS custom properties, next/font, TypeScript. No test framework exists in this repo; verification is `tsc --noEmit`, dual-brand `next build`, and runtime curl/visual checks — standing up a unit-test harness for a CSS/config relocation is YAGNI.

**Working directory for all commands:** `/Users/tirtharajbhowmik/Desktop/Ads Research/website-ui-spec/web` (the git repo root is one level up).

**Known scope notes (from the approved spec):**
- Homepage section structure stays; only tokens/fonts/config swap. The PDF's white footer / other page types are later stretches.
- The uncommitted `src/app/layout.tsx` metadata change in the working tree gets absorbed by Task 4.

- [ ] **Step 1: Write the generator script**

Create `web/scripts/select-brand.mjs`:

```js
// Writes src/brand-active/* as re-exports of the brand chosen by $BRAND.
// Run before dev/build/typecheck: `BRAND=quaeris node scripts/select-brand.mjs`
import { mkdirSync, writeFileSync } from "node:fs";
import path from "node:path";
import { fileURLToPath } from "node:url";

const VALID = ["veris", "quaeris"];
const brand = process.env.BRAND ?? "veris";
if (!VALID.includes(brand)) {
  console.error(`Unknown BRAND "${brand}". Valid brands: ${VALID.join(", ")}`);
  process.exit(1);
}

const here = path.dirname(fileURLToPath(import.meta.url));
const dir = path.join(here, "..", "src", "brand-active");
mkdirSync(dir, { recursive: true });

const banner = "GENERATED by scripts/select-brand.mjs — do not edit";
writeFileSync(path.join(dir, "config.ts"), `// ${banner}\nexport * from "../brands/${brand}/config";\n`);
writeFileSync(path.join(dir, "fonts.ts"), `// ${banner}\nexport * from "../brands/${brand}/fonts";\n`);
writeFileSync(path.join(dir, "tokens.css"), `/* ${banner} */\n@import "../brands/${brand}/tokens.css";\n`);
console.log(`brand-active → ${brand}`);
```
