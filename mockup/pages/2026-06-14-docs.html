Quick Start

3 steps to your first
governed answer.

From zero to a traced, auditable query result in minutes.

Connect your data warehouse

Quaeris is warehouse-native — your data never moves. Provide credentials for Snowflake, BigQuery, Redshift, or Databricks and Quaeris introspects your schema automatically.

Snowflake connector guide BigQuery setup All supported connectors
# quaeris.yaml — warehouse connection   warehouse:   type: "snowflake"   account: "your-account"   database: "PROD_ANALYTICS"   schema: "finance"   role: "QUAERIS_READER"   # Credentials resolved from env — never stored in config   credentials: "env://SNOWFLAKE_CREDS" Schema introspected · data stays in warehouse

Define a certified metric

In the semantic layer editor, declare your metric once — formula, GL accounts, owner, and certification date. Every query that touches "revenue" will use this exact definition forever.

Metric definition syntax Versioning & certification workflow Semantic layer overview
# semantic/metrics/revenue.yaml   metric:   name: "revenue"   version: "3.1"   formula: "SUM(gl_4000_4099) - returns"   gl_accounts: "4000–4099"   owner: "controller@company.com"   certified_at: "2026-06-01"   access:     roles: ["finance", "executive"] Version-locked · RBAC enforced at query time

Ask your first question

Use the Query API, the chat UI, or an MCP tool call to ask a natural-language question. Quaeris returns a certified answer with full lineage — traceable back to the source table and GL account.

Query API reference Understanding lineage in results MCP query_governed tool
# POST /v1/query   question: "What was Q3 revenue by region?" context: { "fiscal_year": 2026 }   # Response includes lineage block: lineage:   metric: "revenue · v3.1"   gl_accounts: "4000–4099"   traced_in_ms: 420   access_enforced: true Audit trail attached · zero hallucination risk
